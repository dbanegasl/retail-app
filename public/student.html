Práctica de Laboratorio 1

Control de Versiones y Capa de Calidad (QA)

Preparemos la arquitectura de nuestro proyecto Data-Driven Retail para soportar validaciones, Unit Tests, y asegurar el código en la nube con Git y GitHub.

1 Estructura de Proyecto Profesional

No más archivos sueltos. Construye esta estructura exacta:

retail-app/ ├── public/ │ ├── index.html │ ├── css/ │ │ └── styles.css │ └── js/ │ └── main.js ├── tests/ │ ├── html.test.js │ └── logic.test.js └── README.md

2 Creando el Código Base

Abre public/index.html y agrega:

HTML
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Data-Driven Retail</title>
    <link rel="stylesheet" href="css/styles.css">
</head>
<body>
    <h1 id="titulo">Bienvenido a Data-Driven Retail</h1>
    <button id="btn-estado">Ver Estado</button>
    <script src="js/main.js"></script>
</body>
</html>
JavaScript
document.getElementById('btn-estado').addEventListener('click', () => {
    alert('Sistemas operativos y estables.');
});

Prueba con Live Server haciendo clic derecho en el HTML.

3 Control de Código con Git

Git Hooks permiten ejecutar tests automáticamente antes de cada commit. Por ahora, iniciamos manualmente:

# Inicializa el repositorio
git init

# Prepara todos los archivos
git add .

# Primer commit
git commit -m "chore: estructura profesional y capa de calidad"

4 Ramas (Branches)

Simula una nueva función sin riesgo de romper main:

# Crea y cambia a nueva rama
git checkout -b feature-darkmode
CSS — Modo Oscuro
body { 
    background-color: #333; 
    color: white; 
}
# Registra el cambio
git add .
git commit -m "style: agrega modo oscuro experimental"

# Regresa a main (el modo oscuro desaparece)
git checkout main

# Fusiona la nueva característica
git merge feature-darkmode

5 Subiendo a GitHub

El repositorio remoto es la única fuente de la verdad (GitOps):

  1. Crea un repositorio público retail-app en GitHub (sin README ni .gitignore)
  2. Conecta y sube:
git branch -M main
git remote add origin https://github.com/TU_USUARIO/retail-app.git
git push -u origin main
